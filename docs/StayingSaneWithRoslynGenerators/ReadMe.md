# Read Me for the Roslyn generator notes

This is currently what I am using instead of slides in my Techorama presentation on Roslyn generators. That talk is a set of strategic for creating a generator that allow you to focus on _your code and generation_ instead of the _generator_. As I gauge interest in this material I may add more commentary.

The generator is one of eight steps, probably the one that will take the least time for most generators. The more time invested in the initial prototype, the easier faster creating a production generator will be. The second most time consuming part is testing, because there are many types.

Enjoy! And feel free to add feedback in the [discussion](https://github.com/KathleenDollard/DragonFruit2/discussions/categories/roslyn-incremental-source-generators). I and others will try to answer questions there, but it can be quite difficult to sort though generator challenges.

The opposite of sanity is an unhappy level of frustration and possible failure.
